ECMAScript

ECMAScript es la especificación donde se mencionan todos los detalles de cómo debe funcionar y comportarse Javascript. De esta forma, los diferentes navegadores (Chrome, Firefox, Opera, Edge, Safari...) saben como deben desarrollar los motores de Javascript para que cualquier código o programa funcione exactamente igual, independientemente del navegador que se utilice.

ECMAScript suele venir acompañado de un número que indica la versión o revisión de la que hablamos (algo similar a las versiones de un programa). En cada nueva versión de ECMAScript, se añaden nuevas funcionalidades, manteniendo Javascript vivo y con novedades que lo hacen un lenguaje de programación moderno y cada vez mejor preparado.

Teniendo esto en cuenta, debemos saber que los navegadores web intentan cumplir la especificación ECMAScript al máximo, pero no todos ellos lo consiguen. Por lo tanto, pueden existir ciertas discrepancias. Por ejemplo, pueden existir navegadores que cumplan la especificación ECMAScript al 80% y otros que sólo la cumplan al 60%. Esto significa que pueden haber características que no funcionen en un navegador específico (y en otros sí).

Además, todo esto va cambiando a medida que se van lanzando nuevas versiones de los navegadores web, donde su compatibilidad ECMAScript suele aumentar.

Versiones de ECMAScript

A lo largo de los años, Javascript ha ido sufriendo modificaciones que los navegadores han ido implementando para acomodarse a la última versión de ECMAScript cuanto antes. La lista de versiones de ECMAScript aparecidas hasta el momento son las siguientes.

El pasado de Javascript

La siguiente lista de versiones las podemos considerar el pasado de Javascript:

  • ECMAScript 1 (1997): Primera versión de ECMAScript.
  • ECMAScript 2 (1998): Versión de mantenimiento.
  • ECMAScript 3 (1999): Se añadieron muchas funcionalidades y se corrigieron errores. RegExp, try/catch...
  • ECMAScript 4 (2008): Nunca fue lanzada oficialmente, ya que se consideró demasiado ambiciosa.
  • ECMAScript 5 (2009): Se añadieron muchas funcionalidades y se corrigieron errores. Se añadió el modo estricto.Strict mode, JSON, etc...
  • ECMAScript 5.1(2011): Cambios leves

El presente de Javascript

La siguiente lista de versiones las podemos considerar el presente de Javascript:

  • ECMAScript 2015 (2015): Se añadió la sintaxis de clases, let, const, arrow functions, template strings, destructuring, promises...
  • ECMAScript 2016 (2016): Se añadieron las funciones Array.prototype.includes y exponentiation operator.
  • ECMAScript 2017 (2017): Se añadieron async/await y Object.values/Object.entries.
  • ECMAScript 2018 (2018): Se añadieron Promise.prototype.finally y rest/spread properties.
  • ECMAScript 2019 (2019): Se añadieron Array.prototype.flat/flatMap y Object.fromEntries.
  • ECMAScript 2020 (2020): Se añadieron nullish coalescing operator y optional chaining.
  • ECMAScript 2021 (2021): Se añadieron String.prototype.replaceAll y Promise.any.
  • ECMAScript 2022 (2022): Se añadieron class fields y top-level await.
  • ECMAScript 2023 (2023): Se añadieron Array.prototype.findLast y Array.prototype.findLastIndex.
  • ECMAScript 2024 (2024): Se añadieron Array.prototype.groupBy y WeakRefs.

    Sexta version de ECMAScript

    La sexta versión de ECMAScript (ECMAScript 2015 o ES6) fue lanzada en 2015 y es una de las versiones más importantes del lenguaje. Introdujo una serie de características y mejoras que sentaron las bases para el desarrollo moderno de JavaScript.

    Características principales de ES6

    • LET Y CONSTlet y const son nuevas formas de declarar variables, ofreciendo una mayor flexibilidad y control sobre el alcance de las variables en comparación con var. const define variables que no pueden ser reasignadas después de la declaración.
    • Funciones de flecha:Las funciones de flecha (arrow functions) proporcionan una sintaxis más concisa para definir funciones.
    • Literales de plantilla:Permiten crear cadenas de texto más fácilmente, integrando expresiones directamente dentro de la cadena.
    • ClasesES6 introduce una sintaxis para clases, facilitando la organización del código y la implementación de la programación orientada a objetos.
    • Desestructuración:Permite extraer valores de objetos y arreglos de manera más sencilla.
    • Módulos:ES6 introduce módulos nativos, lo que facilita la organización y reutilización del código en proyectos más grandes.
    • Mapas y Conjuntos: Ofrecen nuevas estructuras de datos para trabajar con pares clave-valor y colecciones de valores únicos.
    • Promesas:Ayudan a manejar operaciones asíncronas de forma más limpia y organizada, evitando el callback hell.

    En resumen, ES6 ha revolucionado JavaScript al introducir una sintaxis más moderna y características que mejoran la eficiencia y la legibilidad del código, lo que la convierte en un estándar fundamental para el desarrollo web y de aplicaciones.

    Un ejemplo de código que utiliza una característica de ES8 es la función Object.entries(). Este método devuelve un array de arrays, donde cada sub-array contiene una clave y el valor correspondiente de un objeto.

    Detalles adicionales sobre ES8

    • Fecha de lanzamientoJunio 2017

      Uso de Object.entries() para iterar sobre un objeto:

      
                const objeto = { a: 1, b: 2, c: 3 };
