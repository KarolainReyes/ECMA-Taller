La especificación ECMAScript

Normas y reglas del funcionamiento de JavaScript

ECMAScript es la especificación donde se mencionan todos los detalles de cómo debe funcionar y comportarse Javascript. De esta forma, los diferentes navegadores (Chrome, Firefox, Opera, Edge, Safari...) saben como deben desarrollar los motores de Javascript para que cualquier código o programa funcione exactamente igual, independientemente del navegador que se utilice.

Versiones de ECMAScript

ECMAScript suele venir acompañado de un número que indica la versión o revisión de la que hablamos (algo similar a las versiones de un programa). En cada nueva versión de ECMAScript, se añaden nuevas funcionalidades, manteniendo Javascript vivo y con novedades que lo hacen un lenguaje de programación moderno y cada vez mejor preparado.

ES1

La primera versión de ECMAScript, conocida como ES1 o ECMAScript 1, fue lanzada en 1997. Esta versión estandarizó JavaScript, un lenguaje de scripting muy popular, y estableció las bases fundamentales para futuras versiones.

Detalles adicionales sobre ES1

  • Características clave: Sintaxis básica, tipos de datos, estructuras de control, funciones y objetos.
  • Objetivo: Proporcionar un lenguaje de scripting interoperable para navegadores web.
  • Impacto: Estableció las bases para el desarrollo web y la programación del lado del cliente.
  • Compatibilidad: La mayoría de los navegadores modernos son compatibles con ES1, aunque algunas características pueden no ser compatibles en versiones más antiguas.
  • Legado: ES1 sentó las bases para el desarrollo de versiones posteriores de ECMAScript, como ES2, ES3 y más allá.
  • Ejemplo de código: Un simple programa que muestra un mensaje en la consola.
    • console.log("Hola, mundo!");

ES2

La segunda versión de ECMAScript (ECMAScript 2 o ES2) fue publicada en 1998 y, aunque no introdujo cambios funcionales significativos, estandarizó las especificaciones técnicas para alinearlas con el estándar ISO/IEC 16262. La segunda version fue un paso a la estandarización del lenguaje, aunque sin grandes novedades funcionales

Detalles adicionales sobre ES1

  • Características clave: Se realizaron correcciones y mejoras menores.
  • Objetivo: Alinear la especificación con el estándar ISO/IEC 16262.
  • Impacto: Mejoró la interoperabilidad entre diferentes implementaciones de ECMAScript.
  • Compatibilidad: La mayoría de los navegadores modernos son compatibles con ES2, aunque no hay características nuevas que destacar.
  • Ejemplo de código: No se introdujeron nuevas características, por lo que el código sigue siendo similar al de ES1.
    • 
                                console.log("Hola, mundo!");

ES4

La cuarta versión de ECMAScript (ECMAScript 4 o ES4) fue un intento ambicioso de mejorar el lenguaje, pero nunca se lanzó oficialmente debido a desacuerdos en la comunidad sobre su complejidad y alcance. Se consideró demasiado ambiciosa y se abandonó en favor de un enfoque más gradual.

Detalles adicionales sobre ES1

  • Fecha de lanzamiento: Nunca se lanzó oficialmente.
  • Características clave: Se propusieron muchas características nuevas, como clases, tipos de datos estáticos y módulos.
  • Objetivo: Mejorar la funcionalidad y la interoperabilidad del lenguaje.
  • Impacto: Aunque nunca se lanzó, influyó en el desarrollo de versiones posteriores de ECMAScript.
  • Compatibilidad: No hay compatibilidad, ya que nunca se lanzó oficialmente.

ES5

La quinta versión de ECMAScript (ECMAScript 5 o ES5) fue lanzada en 2009 y es una de las versiones más importantes del lenguaje. Introdujo una serie de características y mejoras que sentaron las bases para el desarrollo moderno de JavaScript.

Detalles adicionales sobre ES1

  • Características clave: Introducción del modo estricto (strict mode), JSON, métodos de array como forEach, map, filter, reduce y mejoras en la manipulación de objetos.
  • Objetivo: Mejorar la funcionalidad y la interoperabilidad del lenguaje.
  • Impacto: Se convirtió en la base para muchas aplicaciones web y bibliotecas populares.
  • Ejemplo de código: Uso del modo estricto y métodos de array.
    • 
                            "use strict";
                            var arr = [1, 2, 3];
                            arr.forEach(function(item) {
                            console.log(item);
                            });
Características de ES5:

ES5 introdujo características como:

  • Modo Estricto: Permite una detección más rigurosa de errores durante la ejecución del código.
  • JSON: Introducción del formato JSON (JavaScript Object Notation) para el intercambio de datos.
  • Métodos de Array: Se añadieron métodos como forEach, map, filter y reduce para facilitar la manipulación de arrays.
  • Mejoras en Objetos: Se introdujeron métodos como Object.keys y Object.create para mejorar la manipulación de objetos.

ES6/ES2015

JavaScript ES6, también conocido como ECMAScript 2015, es la sexta versión del estándar de JavaScript. Fue lanzado en junio de 2015 y trajo consigo una serie de mejoras y características nuevas para el lenguaje. Estas mejoras están diseñadas para hacer que el desarrollo de aplicaciones web sea más eficiente y fácil de leer y mantener.

Detalles adicionales sobre ES1

  • LET Y CONSTlet y const son nuevas formas de declarar variables, ofreciendo una mayor flexibilidad y control sobre el alcance de las variables en comparación con var. const define variables que no pueden ser reasignadas después de la declaración.
  • Funciones de flecha:Las funciones de flecha (arrow functions) proporcionan una sintaxis más concisa para definir funciones.
  • Literales de plantilla:Permiten crear cadenas de texto más fácilmente, integrando expresiones directamente dentro de la cadena.
  • ClasesES6 introduce una sintaxis para clases, facilitando la organización del código y la implementación de la programación orientada a objetos.
  • Desestructuración:Permite extraer valores de objetos y arreglos de manera más sencilla.
  • Módulos:ES6 introduce módulos nativos, lo que facilita la organización y reutilización del código en proyectos más grandes.
  • Mapas y Conjuntos: Ofrecen nuevas estructuras de datos para trabajar con pares clave-valor y colecciones de valores únicos.
  • Promesas:Ayudan a manejar operaciones asíncronas de forma más limpia y organizada, evitando el callback hell.

En resumen, ES6 ha revolucionado JavaScript al introducir una sintaxis más moderna y características que mejoran la eficiencia y la legibilidad del código, lo que la convierte en un estándar fundamental para el desarrollo web y de aplicaciones.

Detalles adicionales sobre ES1

Ejemplo de código:

Uso de Array.prototype.flat:

En este ejemplo, flat() aplana el array multidimensional en un array unidimensional.

ES12/ES2021

La duodécima versión de ECMAScript, también conocida como ES2021, se publicó en junio de 2021. Las principales novedades incluyen el método String.prototype.replaceAll, Promise.any y WeakRefs.

Detalles adicionales sobre ES1

  • String.prototype.replaceAll:Permite reemplazar todas las ocurrencias de una subcadena en una cadena.
  • Promise.any:Devuelve la primera promesa que se resuelve, ignorando las rechazadas.
  • WeakRefs:Permite crear referencias débiles a objetos, lo que facilita la gestión de memoria.
Ejemplo de código:

Uso de String.prototype.replaceAll:


                                const cadena = "Hola, mundo. Hola, universo.";
                                const nuevaCadena = cadena.replaceAll("Hola", "Adiós");
                                console.log(nuevaCadena); // "Adiós, mundo. Adiós, universo."

En este ejemplo, replaceAll() reemplaza todas las ocurrencias de "Hola" por "Adiós".

Detalles adicionales sobre ES1

  • Sintaxis de campos de clase:Permite definir campos y métodos dentro de una clase.
  • Top-level await:Permite usar await en el nivel superior de un módulo.
Ejemplo de código:

Uso de la sintaxis de campos de clase:


                                    class Persona {
                                    nombre = "Juan";
                                    saludar() {
                                        console.log(`Hola, soy ${this.nombre}`);

                                    const persona = new Persona();

En este ejemplo, se define un campo nombre y un método saludar() dentro de la clase Persona.

ES14/ES2023

La decimocuarta versión de ECMAScript, también conocida como ES2023, se publicó en junio de 2023. Las principales novedades incluyen el método Array.prototype.findLast y Array.prototype.findLastIndex.

Detalles adicionales sobre ES1

  • Array.prototype.findLast:Devuelve el último elemento que cumple con una condición dada.
  • Array.prototype.findLastIndex:Devuelve el índice del último elemento que cumple con una condición dada.
Ejemplo de código:

Uso de Array.prototype.findLast:


                                    const array = [1, 2, 3, 4, 5];
                                    const ultimoPar = array.findLast(num => num % 2 === 0);
                                    console.log(ultimoPar); // 4

En este ejemplo, findLast() devuelve el último número par del array.

ES15/ES2024

La decimoquinta versión de ECMAScript, también conocida como ES2024, se publicará en junio de 2024. Las principales novedades incluyen el método Array.prototype.groupBy y WeakRefs.

Detalles adicionales sobre ES1

  • Array.prototype.groupBy:Permite agrupar elementos de un array según una función de agrupamiento.
  • WeakRefs:Permite crear referencias débiles a objetos, lo que facilita la gestión de memoria.
Ejemplo de código:

Uso de Array.prototype.groupBy:


                                    const array = [1, 2, 3, 4, 5];
                                    const grupos = array.groupBy(num => num % 2 === 0 ? "par" : "impar");

En este ejemplo, groupBy() agrupa los números en pares e impares.
